  2. Create `src/app/render/client.tsx` (Client Component with dynamic ssr:false loading RenderApp):
     ```typescript
     'use client';

     import dynamic from 'next/dynamic';

     const RenderApp = dynamic(() => import('../../RenderApp'), { ssr: false });

     export function RenderClient() {
       return <RenderApp />;
     }
     ```

     This works because:
     - The export service's Puppeteer calls `evaluateOnNewDocument` to set `window.__EXPORT_CONFIG__` BEFORE navigation
     - RenderApp.tsx reads `window.__EXPORT_CONFIG__!` directly (line 14)
     - The /render route always loads RenderApp, which always reads the config
     - No conditional routing needed -- the /render route IS the routing

  3. Update `export-service/src/shared/config.ts`:
     - Change the frontendUrl default to point to the Next.js dev server's /render route
     - Current: `frontendUrl: process.env.FRONTEND_URL ?? 'http://localhost:${port}'`
     - New: `frontendUrl: process.env.FRONTEND_URL ?? 'http://localhost:3000/render'`

     The default now points to Next.js on port 3000 (not the export service on 3001) with the /render path. In production, FRONTEND_URL env var overrides this. Also update the comment from "Vite dev server" to "Next.js dev server".

  4. Also update the `frontendDistPath` comment in config.ts to note that this path is no longer used in dev (Next.js serves its own files), but may be relevant for production static export in the future. The path itself can remain for now as it won't break anything.
