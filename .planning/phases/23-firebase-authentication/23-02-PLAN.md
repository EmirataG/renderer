  <action>
    1. Create `proxy.ts` in the project root (Next.js 16 file convention, replaces middleware.ts):
       - Import `NextResponse` from `next/server` and `NextRequest` type from `next/server`
       - Define `PUBLIC_PATHS = ['/login', '/render', '/api/auth']`
       - Export `function proxy(request: NextRequest)`:
         - Get `pathname` from `request.nextUrl`
         - If `pathname` matches any PUBLIC_PATHS prefix (using `.some(p => pathname.startsWith(p))`), return `NextResponse.next()`
         - Get session cookie: `request.cookies.get('__session')`
         - If no session cookie value, redirect to `/login`: `NextResponse.redirect(new URL('/login', request.url))`
         - Otherwise, `NextResponse.next()`
       - Export `config` with `matcher: ['/((?!_next/static|_next/image|favicon.ico).*)']`
       - This is a LIGHTWEIGHT check -- only verifies cookie existence, does NOT verify JWT signature (that would add latency to every request). Full verification happens in Route Handlers when needed.

    2. Modify `src/app/[[...slug]]/client.tsx`:
       - Keep existing `'use client'` directive and dynamic import of App
       - Add imports: `signOut` from `firebase/auth`, `auth` from `@/lib/firebase-client`, `useRouter` from `next/navigation`
       - Add `handleSignOut` async function: calls `fetch('/api/auth/session', { method: 'DELETE' })` to clear server cookie, then `signOut(auth)` to clear client state, then `router.push('/login')`
       - Add a sign-out button positioned in the top-right corner: `position: fixed`, `top: 12px`, `right: 12px`, `z-index: 50`. Style: small subtle button with Tailwind classes `fixed top-3 right-3 z-50 px-3 py-1.5 text-xs bg-neutral-800 text-neutral-300 rounded hover:bg-neutral-700 transition-colors`. Text: "Sign out".
       - The button should be rendered OUTSIDE the dynamic App component, alongside it in the ClientOnly return JSX (so it renders immediately, not after App loads).
  </action>

    - proxy.ts does NOT import firebase-admin (lightweight check only)
  </verify>
