Create a new Zustand store for caching musical events. Follow the existing syncStore.ts pattern.

**CachedEvent interface:**
```typescript
export interface CachedEvent {
  id: string;            // e.g., "evt-0", "evt-1"
  beatOnset: number;     // Quarter-note-based timing (RealValue)
  beatDuration: number;  // Duration until next event
  svgIds: string[];      // Verovio note xml:id values
  pageIndex: number;     // 0-based page index
  globalY: number;       // Y position in global coordinate space
}
```

**Store state:**
- `events: CachedEvent[]` - The cached event array
- `svgPagesRef: string[] | null` - Reference for invalidation check
- `eventById: Map<string, CachedEvent>` - O(1) lookup by event ID
- `eventsByPage: Map<number, CachedEvent[]>` - Events grouped by page

**Store actions:**
- `setEvents(events, svgPagesRef)` - Store events and build lookup indices
- `invalidate()` - Clear all cached data

Build lookup indices inside setEvents (not in selectors). Use `new Map()` for each index to ensure proper reference comparison.

Export: `useEventStore` (the hook) and `CachedEvent` (the type)

  <name>Task 2: Refactor getEvents.ts with two-phase extraction functions</name>
  <files>src/lib/getEvents.ts</files>
  <action>
Add two new functions to getEvents.ts that split extraction into pure data (timemap) and DOM-dependent (Y positions) phases. Keep getEventsFromVerovio for backward compatibility.

**TimemapEvent interface (intermediate type):**
```typescript
export interface TimemapEvent {
  id: string;
  beatOnset: number;
  beatDuration: number;
  svgIds: string[];
}
```

**Function 1: extractTimemapEvents(toolkit)**
- Takes Verovio toolkit only (no DOM needed)
- Calls `toolkit.renderToTimemap()`
- Filters to onset entries (entries with `on` array)
- Maps to TimemapEvent objects with id, beatOnset (qstamp/4), svgIds
- Calculates beatDuration for each event
- Returns `TimemapEvent[]`

**Function 2: computeEventPositions(timemapEvents, toolkit, pageContainers, pageOffsets)**
- Takes timemap events + DOM references
- For each event:
  - Get pageIndex via `toolkit.getPageWithElement(svgIds[0])` (1-based, convert to 0-based)
  - Find the page container at pageIndex
  - Query for note element by ID
  - Find parent g.system element
  - Compute localY from system center relative to container
  - Compute globalY as `pageOffsets[pageIndex] + localY`
- Returns `CachedEvent[]` (import from eventStore.ts)

**Backward compatibility:**
Keep getEventsFromVerovio unchanged. It already has the two paths (page-aware and single-container). The new functions are used by RegularRenderer directly; SyncEditor continues using getEventsFromVerovio with single container.
