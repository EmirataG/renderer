**export-service/src/server.ts** -- add static serving for standalone page:

1. Add path import: `import { join } from 'node:path';` (if not already imported -- check first, currently uses `import.meta.dirname` indirectly via config).

2. Register @fastify/static for standalone-dist AFTER the existing fastifyStatic registration (which serves frontendDistPath). Use `decorateReply: false` (the existing registration already uses this, so both can coexist):
   ```typescript
   // Serve standalone render page assets (animation bundle + verovio WASM)
   await server.register(fastifyStatic, {
     root: join(import.meta.dirname, '../standalone-dist'),
     prefix: '/static/',
     decorateReply: false,
   });
   ```

3. Add a GET /render route that serves render.html:
   Since both @fastify/static registrations use `decorateReply: false`, `reply.sendFile()` is NOT available as a decorator method. Use the manual approach -- read the file at startup and serve it:
   ```typescript
   import { readFileSync } from 'node:fs';
   // ... at top of main():
   const renderHtml = readFileSync(join(import.meta.dirname, '../standalone-dist/render.html'), 'utf-8');
   // ... after static plugin registration, before API routes:
   server.get('/render', async (_request, reply) => {
     return reply.type('text/html').send(renderHtml);
   });
   ```
   Place the /render route AFTER the static plugin registration and BEFORE the API routes registration.

4. Update the existing frontendDistPath static registration comment to note it is legacy (Vite-era) and may be removed in the future. Keep it for now to avoid breaking anything.

- FIX: Change to `const frontendUrl = config.frontendUrl;` -- no trailing slash. The config already has the full path including /render.
