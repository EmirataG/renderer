**render.ts** -- standalone page entry point. Reads `window.__EXPORT_CONFIG__` (ExportConfig from pageSetup.ts), initializes Verovio, renders score, extracts events, computes positions, sets up animation controller, and signals readiness.

Full implementation sequence (read __EXPORT_CONFIG__ synchronously at top):

1. **Read config**: `const config = (window as any).__EXPORT_CONFIG__` -- same ExportConfig shape from export-service/src/browser/pageSetup.ts.

2. **Constants**: `const EDITOR_WIDTH = 980; const scaleFactor = config.viewportWidth / EDITOR_WIDTH;`

3. **Build DOM structure** matching RegularRenderer's render output:
   - Outer container: `width: viewportWidth, height: viewportHeight, overflow: hidden`
   - Scale wrapper: `transform-origin: top left, transform: scale(scaleFactor)` (matches RenderApp.tsx line 34)
   - Background div: `width: EDITOR_WIDTH (after scale), height: viewportHeight/scaleFactor, backgroundImage if bgUrl, backgroundSize: cover`
   - Score region positioned absolutely: `left: scoreRegion.x, top: scoreRegion.y, width: scoreRegion.width, height: scoreRegion.height, overflow: hidden` (or full container if no scoreRegion)
   - Camera div (for translateY animation): `display: flex, width: 100%, pointerEvents: none, transition: none`
   - Score div (holds SVG pages): `width: regionWidth, lineHeight: 0, fontSize: 0`

4. **Score color CSS**: Generate a `<style>` element with the same CSS as RegularRenderer lines 285-319. The pattern is: `.preview-score { color: {scoreColor} }` plus `.preview-score path, .preview-score line, ...` selectors with `fill: currentColor` / `stroke: currentColor`. Also hide labels if `config.hideLabels`: `.preview-score .label { display: none }`.

5. **Verovio init** (async, inside a main() function):
   - `const toolkit = new (window as any).verovio.toolkit()` -- UMD build uses global
   - Set options matching useVerovio.ts lines 106-119:
     ```
     const scoreWidth = config.scoreRegion?.width ?? EDITOR_WIDTH;
     const verovioScale = Math.round(40 * (config.scoreScale ?? 1));
     toolkit.setOptions({ font: config.musicFont || 'Bravura', fontLoadAll: true,
       pageWidth: (scoreWidth * 100) / verovioScale, pageHeight: 2970, scale: verovioScale,
       adjustPageHeight: true, pageMarginTop: 0, pageMarginBottom: 0,
       svgViewBox: true, svgRemoveXlink: true, breaks: 'auto', header: 'none', footer: 'none' });
     ```
   - `toolkit.loadData(config.musicXml)` -- throw on false
   - `toolkit.renderToMIDI()`

6. **Render pages**: Loop `toolkit.getPageCount()`, call `toolkit.renderToSVG(i)` for each page. Apply `trimPageTopMargin` for pages 2+ (port the regex logic from useVerovio.ts lines 40-66 inline or as a local function). Extract page heights using regex (port `extractPageHeight` from useVerovio.ts lines 23-30). Compute `pageOffsets` (cumulative heights) and `totalHeight`.

7. **Mount SVGs in DOM**: Create a div per page inside the score div, set `innerHTML = svg`. Store page container references in an array.

8. **Extract events**: Port `extractTimemapEvents` logic inline (it only needs `toolkit.renderToTimemap()` -- filter to onset entries, map to `{id, beatOnset, beatDuration, svgIds}`). Then interpolate timestamps using the same algorithm from `src/lib/interpolation.ts` -- port the `interpolateTimestamps` function inline (it's pure math, ~80 lines). Pass `config.syncAnchors` converted to a Map: `new Map(Object.entries(config.syncAnchors))`.

9. **Compute event positions**: Port `computeEventPositions` logic (from src/lib/getEvents.ts lines 196-250). For each event: use `toolkit.getPageWithElement(svgId)` to find page, then measure `g.system` center Y within page container, add pageOffset. Detect CSS scale via `getBoundingClientRect().width / clientWidth` on first container (the scaleFactor wrapper means positions need division by domScale).

10. **Merge Y positions**: Map interpolated events + globalY from position computation. The final events array has `{ computedTimestamp, y: globalY, svgIds }`.

11. **Setup borders**: If `config.scoreBorder !== 'none'`, import and call `generateBorderSvg` from borders.ts for top and bottom positions. Insert as `innerHTML` in absolutely-positioned divs matching RegularRenderer lines 920-970.

12. **Create animation state**: `const state = createAnimationState()`.

13. **Build AnimationConfig** from ExportConfig:
    ```
    { scoreColor: config.scoreColor,
      activeNoteheadColor: config.activeNoteheadColor ?? config.scoreColor,
      activeNoteheadScale: config.activeNoteheadScale ?? 1,
      activeNoteheadHoldMs: config.activeNoteheadHoldMs ?? 200,
      activeNoteheadExitMs: config.activeNoteheadExitMs ?? 200,
      colorFullNote: config.colorFullNote ?? false,
      scoreRegionHeight: config.scoreRegion?.height ?? null,
      containerHeight: config.viewportHeight / scaleFactor,
      totalHeight }
    ```

14. **Expose animationController on window**:
    ```
    (window as any).animationController = {
      setFrame: (frameNumber: number, fpsValue: number = 30) => {
        setTimestamp(frameNumber / fpsValue, events, state, animConfig, cameraEl, scoreEl);
      },
      setTimestamp: (seconds: number) => {
        setTimestamp(seconds, events, state, animConfig, cameraEl, scoreEl);
      },
      getDuration: () => config.audioDuration,
      getFps: () => config.fps,
    };
    ```

15. **Signal readiness**: `(window as any).rendererReady = true;` -- ONLY after all steps complete.

Add console.log at each major step for debugging (Verovio init, page render, event extraction, position computation, controller exposed, ready).

**CRITICAL**: The `containerHeight` in AnimationConfig must be `config.viewportHeight / scaleFactor` (pre-scale coordinates), because the camera translateY operates inside the scale wrapper. Similarly, scoreRegionHeight should use the raw scoreRegion.height value from config (which is in pre-scale coordinates since config already specifies at viewport level -- but verify: if scoreRegion coordinates are in viewport pixels, they need dividing by scaleFactor too; if they're in editor-space pixels, they're fine as-is). Look at how RenderApp.tsx passes scoreRegion to RegularRenderer to determine this.

**build-standalone.mjs** -- esbuild build script at `export-service/build-standalone.mjs`:
```javascript
import { build } from 'esbuild';
import { copyFileSync, mkdirSync } from 'fs';
import { dirname, join } from 'path';
import { fileURLToPath } from 'url';

const __dirname = dirname(fileURLToPath(import.meta.url));

mkdirSync(join(__dirname, 'standalone-dist'), { recursive: true });

await build({
  entryPoints: [join(__dirname, 'src/standalone/render.ts')],
  bundle: true,
  format: 'iife',
  outfile: join(__dirname, 'standalone-dist/render-bundle.js'),
  platform: 'browser',
  target: 'chrome120',
});

copyFileSync(
  join(__dirname, '../node_modules/verovio/dist/verovio-toolkit-wasm.js'),
  join(__dirname, 'standalone-dist/verovio-toolkit-wasm.js')
);
copyFileSync(
  join(__dirname, 'src/standalone/render.html'),
  join(__dirname, 'standalone-dist/render.html')
);

console.log('Standalone build complete -> standalone-dist/');
```
