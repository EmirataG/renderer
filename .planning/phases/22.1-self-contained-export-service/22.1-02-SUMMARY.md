## Accomplishments
- Wired standalone-dist/ into Fastify with @fastify/static for /static/ prefix and manual /render route

- Fixed trailing slash issue in jobManager.ts that would have broken Puppeteer navigation
- Deleted frontend /render route (page.tsx, client.tsx), RenderApp.tsx, and removed /render from proxy.ts PUBLIC_PATHS

## Decisions Made
- Used readFileSync at startup + reply.send for the /render route since both @fastify/static registrations use decorateReply: false.
- Fixed verovio initialization by importing from verovio/wasm and verovio/esm (same pattern as frontend's verovioService.ts) instead of loading the Node.js UMD build via script tag.

## Deviations from Plan
- Verovio initialization approach changed from UMD script tag to ESM imports bundled by esbuild. The UMD build (verovio-toolkit-wasm.js) requires node:fs and node:crypto, making it incompatible with browser environments.
- esbuild format changed from IIFE to ESM to support import.meta.url needed by verovio's WASM loader.

## Issues Encountered
- **verovio UMD crash**: `getToolkitFunction(...) is not a function` — the verovio-toolkit-wasm.js UMD build is Node.js-only. Fixed by switching to ESM imports.
