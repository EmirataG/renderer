  <action>
Create a new SingleLineRenderer.tsx component that displays scores horizontally with smooth camera tracking and notehead animation. Mirror RegularRenderer.tsx structure but adapt for horizontal axis.

**Component structure:**

1. **Props interface** - Same as RegularRenderer:
   - xml, bgUrl, fps, scoreColor
   - syncAnchors, audioUrl
   - scoreRegion, scoreBorder, scoreScale
   - Notehead animation props (activeNoteheadColor, activeNoteheadScale, entryMs, holdMs, exitMs, colorFullNote)

2. **Hooks and state:**
   - Use `useSingleLineVerovio(xml, verovioScale, measuresPerSection)` to get sections, sectionWidths, sectionOffsets, totalWidth, toolkit
   - Use `useEventStore` to read/write cached events
   - Local state for interpolatedEvents, containerWidth/Height, isPlaying, audioDuration
   - Refs: cameraRef (div), scoreRef (div), audioRef, sectionContainerRefs (array)

3. **Layout structure:**
   ```jsx
   <div style={{ overflow: 'hidden', width: viewportWidth, height: viewportHeight }}>
     <div ref={cameraRef} style={{
       display: 'flex',
       flexDirection: 'row',
       transition: 'transform 200ms ease-out'
     }}>
       {sections.map((svg, i) => (
         <div
           key={i}
           ref={el => sectionContainerRefs.current[i] = el}
           style={{ flexShrink: 0, width: sectionWidths[i] }}
           dangerouslySetInnerHTML={{ __html: svg }}
         />
       ))}
     </div>
   </div>
   ```

4. **Event extraction effect:**
   - After sections render, call `extractTimemapEvents(toolkit)` then `computeSectionPositions(events, sectionContainers, sectionOffsets)`
   - Store events in eventStore with `setEventsInStore(eventsWithPositions, sections)`
   - Reset noteheads with `resetNoteheadAnimations(scoreRef.current)`

5. **Interpolation effect:**
   - When events or syncAnchors change, call `interpolateTimestamps(events, syncAnchors)`
   - Merge globalX from events into interpolated results (same pattern as RegularRenderer merges globalY)

6. **Camera function (applyCamera):**
   ```typescript
   function applyCamera(targetX: number) {
     const scoreWidth = totalWidth;
     const viewportWidth = scoreRegion?.width ?? containerWidth;

     // Center the target X in viewport (50% position per CONTEXT.md)
     let cameraX = targetX - viewportWidth / 2;

     // Edge clamping: flush left at start, stop at right edge
     cameraX = Math.max(0, cameraX);
     cameraX = Math.min(cameraX, Math.max(0, scoreWidth - viewportWidth));

     if (cameraRef.current) {
       cameraRef.current.style.transform = `translateX(${-cameraX}px)`;
     }
   }
   ```

7. **Animation loop (animateSync):**
   - Same pattern as RegularRenderer but use `event.globalX` instead of `event.y` for camera position
   - Call `animateNoteheads(scoreRef.current, evt.svgIds, options)` for each new event
   - Call `applyCamera(currentXRef.current)` each frame

8. **Transport controls:**
   - play(), stop(), reset() - identical to RegularRenderer
   - canPlay gating: hasAudio && hasFirstAnchor && hasLastAnchor
   - Transport buttons UI at bottom

9. **Puppeteer support (optional, lower priority):**
   - Expose animationController on window for frame capture (same as RegularRenderer)
   - setTimestamp function that applies camera and animation state directly

**Key differences from RegularRenderer:**
- Use useSingleLineVerovio instead of useVerovio
- Horizontal flexbox layout instead of vertical stacking
- translateX instead of translateY for camera
- globalX instead of globalY for event positions
- sectionContainerRefs instead of pageContainerRefs

**Styling:**
- Include same scoreColorCss as RegularRenderer
- Same preview-score class for SVG styling
- Same border handling (adapted for horizontal if needed)

**Do NOT:**
- Implement virtual scrolling (Phase 13)
- Add toggle UI for switching renderers
- Modify existing RegularRenderer
