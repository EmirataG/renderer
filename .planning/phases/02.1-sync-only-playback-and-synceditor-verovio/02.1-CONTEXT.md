# Phase 2.1: Sync-Only Playback & SyncEditor Verovio - Context

**Gathered:** 2026-02-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Remove BPM-based playback entirely — playback is solely driven by sync anchors and audio interpolation. Migrate SyncEditor from OSMD to Verovio so both views use the same rendering engine. No new features, just removal of BPM mode and engine swap for SyncEditor.

</domain>

<decisions>
## Implementation Decisions

### No-anchors state
- Play button is disabled (grayed out) until both audio is uploaded AND first and last events have sync anchors set
- Audio is always required for playback — no anchors-only silent mode
- Inline text below transport bar shows context-aware message:
  - No audio: "Upload audio to enable playback"
  - No anchors (but audio exists): "Set first and last sync anchors to enable playback"
  - Ready: no message (or hide the text)

### BPM removal scope
- Remove BPM slider from sidebar UI completely
- Remove all BPM-related code from RegularRenderer: bpm prop, setupEventBPM(), animateBPM(), and all BPM-related refs — clean, total removal
- FPS prop remains configurable (still used for sync animation frame throttling)
- Puppeteer animation controller code stays as-is but won't be used or maintained — deferred cleanup

### SyncEditor interaction
- SyncEditor uses Verovio instead of OSMD — reuse useVerovio hook and getEventsFromVerovio where possible
- SyncEditor and RegularRenderer may need separate toolkit instances since they have separate styles, but use the same patterns
- Note clicking behavior stays identical — click note in SVG to select, set timestamp anchor. Just swap OSMD SVG IDs for Verovio SVG IDs
- SyncEditor does not enforce anchor requirements — that's the Play button's responsibility in RegularRenderer

### Transport controls
- Same three buttons: Play, Pause, Reset — all sync-only
- Play disabled until audio + first/last anchors are set
- End-of-audio behavior unchanged (keep current implementation)
- Context-aware inline text below transport:
  - Different messages depending on what's missing (audio vs anchors)

### Claude's Discretion
- Whether SyncEditor can share a toolkit instance with RegularRenderer or needs its own (depends on style separation needs)
- Exact wording of context-aware transport messages
- How to determine "first and last" events programmatically (first and last in the events array)

</decisions>

<specifics>
## Specific Ideas

- "The score in sync view should behave EXACTLY as it does now, except it should use Verovio"
- Puppeteer code is kept but won't be used — user doesn't want video export
- The two scores (RegularRenderer and SyncEditor) need separate styles — SyncEditor has clickable notes, RegularRenderer has pointer-events: none

</specifics>

<deferred>
## Deferred Ideas

- Puppeteer/video export cleanup — can be removed in Phase 5 (Validation and Cleanup) or later
- VAL-03 (BPM-based animation) requirement needs to be removed from Phase 3 success criteria since BPM is being deleted

</deferred>
