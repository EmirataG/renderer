   - `import { shutdownPool } from './browser/browserPool.js';`

2. Register @fastify/static BEFORE the API routes (after multipart registration):
   ```typescript
   await server.register(fastifyStatic, {
     root: config.frontendDistPath,
     prefix: '/',
     decorateReply: false,
   });
   ```
   `decorateReply: false` avoids conflicts with other plugins that decorate the reply object.

3. Add pool shutdown to the existing `onClose` hook. The current hook clears the cleanup timer. Extend it to also shut down the browser pool:
   ```typescript
   server.addHook('onClose', async () => {
     clearInterval(cleanupTimer);
     await shutdownPool();
   });
   ```
   This ensures Chrome processes are cleaned up when the server stops.

Note: The @fastify/static plugin serves the Vite-built frontend at `/`, so Puppeteer will navigate to `http://localhost:3001/` to load the app. API routes at `/api/*` are registered separately and won't conflict because they have a prefix.
