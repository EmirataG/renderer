Create a new hook following the same pattern as `useVerovio.ts` but for horizontal section-based rendering.

**Interface:**
```typescript
export interface UseSingleLineVerovioResult {
  sections: string[];           // Array of SVG strings, one per section
  sectionWidths: number[];      // Width of each section in pixels
  sectionOffsets: number[];     // Cumulative X offset for each section
  totalWidth: number;           // Total score width (sum of all widths)
  sectionCount: number;         // Number of sections
  measureCount: number;         // Total measures in score
  toolkit: VerovioToolkit | null;
  isLoading: boolean;
  error: string | null;
}
```

**Hook signature:**
```typescript
export function useSingleLineVerovio(
  xml: string,
  scale: number = 40,
  measuresPerSection: number = 15
): UseSingleLineVerovioResult
```

**Implementation details:**

1. **Verovio options for horizontal layout:**
   - `breaks: 'none'` (forces single horizontal system)
   - `pageWidth: 100000` (large width to prevent wrapping)
   - `pageHeight: 100` (minimal, adjustPageHeight expands)
   - `adjustPageHeight: true`
   - All page margins: 0
   - `svgViewBox: true`
   - `svgRemoveXlink: true`
   - `header: 'none'`, `footer: 'none'`

2. **Get measure count:**
   Use `toolkit.getMEI()` and count `<measure ` occurrences with regex:
   ```typescript
   const mei = toolkit.getMEI();
   const measureMatches = mei.match(/<measure /g);
   const measureCount = measureMatches ? measureMatches.length : 0;
   ```

3. **Section rendering loop:**
   ```typescript
   const MEASURES_PER_SECTION = measuresPerSection;
   const sections: string[] = [];

   for (let start = 1; start <= measureCount; start += MEASURES_PER_SECTION) {
     const end = Math.min(start + MEASURES_PER_SECTION - 1, measureCount);
     toolkit.select({ measureRange: `${start}-${end}` });
     toolkit.redoLayout();
     const svg = toolkit.renderToSVG(1); // Always page 1 after select
     sections.push(svg);
   }

   // Clear selection for future operations
   toolkit.select({});
   toolkit.redoLayout();
   ```

4. **Width extraction function:**
   ```typescript
   function extractSectionWidth(svgString: string): number {
     const widthMatch = svgString.match(/width="(\d+(?:\.\d+)?)px"/);
     if (widthMatch) return parseFloat(widthMatch[1]);
     const vbMatch = svgString.match(/viewBox="0 0 ([\d.]+) [\d.]+"/);
     if (vbMatch) return parseFloat(vbMatch[1]);
     return 0;
   }
   ```

5. **Compute widths and offsets:**
   ```typescript
   const widths = sections.map(extractSectionWidth);
   const offsets: number[] = [];
   let cumulative = 0;
   for (const w of widths) {
     offsets.push(cumulative);
     cumulative += w;
   }
   // totalWidth = cumulative
   ```

6. **State management:**
   - Use same useState pattern as useVerovio.ts
   - Use useRef for toolkit
   - Use useEffect with cancellation flag
   - Dependency array: `[xml, scale, measuresPerSection]`

7. **Error handling:**
   - Handle empty/invalid xml gracefully
   - Handle loadData failures
   - Handle edge case: measureCount = 0
