<task type="auto">
  <name>Task 1: Create fly.toml and deploy to Fly.io</name>
  <files>fly.toml</files>
  <action>
Create `fly.toml` at the repo root (`renderer/fly.toml`) with the following configuration:

```toml
app = "manuscript-export"
primary_region = "iad"

[build]
  dockerfile = "Dockerfile"

[env]
  NODE_ENV = "production"
  PORT = "8080"

[http_service]
  internal_port = 8080
  force_https = true
  auto_stop_machines = "stop"
  auto_start_machines = true
  min_machines_running = 0

  [http_service.concurrency]
    type = "requests"
    soft_limit = 1
    hard_limit = 2

[[http_service.checks]]
  grace_period = "30s"
  interval = "30s"
  method = "GET"
  path = "/health"
  timeout = "5s"

[[vm]]
  size = "shared-cpu-2x"
  memory = "2gb"
```

**Key configuration decisions:**
- `primary_region = "iad"` (US East, low latency for initial deployment)
- `auto_stop_machines = "stop"` (stops the machine when idle, restarts on request)
- `min_machines_running = 0` (all machines can stop, zero cost when idle)
- `soft_limit = 1, hard_limit = 2` (conservative concurrency — one export at a time is realistic for 2GB RAM budget; Chrome + FFmpeg + Node.js consume ~1.5GB per export)
- `grace_period = "30s"` (covers cold start: VM boot + Node.js startup + Fastify ready)
- `shared-cpu-2x` with 2GB RAM (Chrome minimum ~1GB, plus FFmpeg + Node.js overhead)
- `dockerfile = "Dockerfile"` (references the Dockerfile at repo root created in Plan 01)

After creating fly.toml, deploy:
1. Run `fly apps create manuscript-export --org personal` (if app doesn't exist yet)
2. Run `fly deploy` from the repo root
3. If deploy fails with auth error, pause and inform user to run `fly auth login`
4. After successful deploy, run `fly status` to verify the machine is running
5. Run `curl https://manuscript-export.fly.dev/health` to verify health check passes
