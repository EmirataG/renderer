  <action>
    Create two new files in `export-service/src/encoding/`.

    **encodeVideo.ts** -- Exports `startVideoEncode(outputPath, fps, width, height)` returning `{ writeFrame, finish }`:

    1. Spawn FFmpeg with args: `-y -f image2pipe -c:v png -framerate {fps} -i pipe:0 -c:v libx264 -pix_fmt yuv420p -preset medium -crf 18 -movflags +faststart -an {outputPath}`
    2. stdio configuration: `['pipe', 'ignore', 'pipe']` (stdin=pipe for frames, stdout=ignore, stderr=pipe for diagnostics)
    3. Collect stderr into a string variable for error reporting (capture last 500 chars on failure)
    4. `writeFrame(buffer: Uint8Array): Promise<void>` -- calls `proc.stdin.write(buffer)`, if return is `false`, awaits `once(proc.stdin, 'drain')` before returning. Import `once` from `node:events`.
    5. `finish(): Promise<void>` -- returns a Promise that: attaches `close` listener (resolve on code 0, reject with stderr on non-zero), attaches `error` listener (reject with spawn error message), then calls `proc.stdin.end()`. CRITICAL: stdin.end() must be called INSIDE the Promise constructor AFTER listeners are attached to avoid missing the close event.
    6. Return type: `{ writeFrame: (buffer: Uint8Array) => Promise<void>; finish: () => Promise<void> }`

    **muxAudio.ts** -- Exports `muxAudio(silentVideoPath, audioPath, outputPath)` and `findAudioFile(tempDir)`:

    1. `findAudioFile(tempDir: string): Promise<string>` -- reads directory with `readdir()`, finds file starting with `audio.`, throws if not found, returns full path via `join(tempDir, audioFile)`.
    2. `muxAudio(silentVideoPath, audioPath, outputPath): Promise<void>` -- spawns FFmpeg with args: `-y -i {silentVideoPath} -i {audioPath} -c:v copy -c:a aac -b:a 192k -shortest -movflags +faststart {outputPath}`. stdio: `['ignore', 'ignore', 'pipe']`. Returns Promise that resolves on exit code 0, rejects with stderr on non-zero. Attaches `error` listener for spawn failures.

    Use `node:child_process` spawn, `node:events` once, `node:fs/promises` readdir, `node:path` join. No npm packages.
    All imports use `.js` extension for ESM NodeNext resolution (matching existing codebase convention).

    2. Replace the frame buffer collection block (lines 125-134, from "// Capture frames" to the `(job as any)` casts) with the FFmpeg pipeline:

       ```
       // Step 1: Start FFmpeg encode process
       this.updateStatus(jobId, 'encoding');
       const silentVideoPath = join(job.tempDir, 'video-silent.mp4');
       const encoder = startVideoEncode(silentVideoPath, exportConfig.fps, viewport.width, viewport.height);

       // Pipe captured frames directly to FFmpeg stdin (bounded memory)
       let frameCount = 0;
       for await (const { buffer } of captureFrames(page, result.totalFrames, exportConfig.fps)) {
         await encoder.writeFrame(buffer);
         frameCount++;
       }

       // Signal end of frames, wait for FFmpeg to finish encoding
       await encoder.finish();

       // Step 2: Mux audio into the silent video
       const audioPath = await findAudioFile(job.tempDir);
       const outputPath = join(job.tempDir, 'output.mp4');
       await muxAudio(silentVideoPath, audioPath, outputPath);

       // Clean up intermediate silent video file
       try { await unlink(silentVideoPath); } catch { /* ignore if already gone */ }

       // Store output path and frame count on job
       job.outputPath = outputPath;
       ```

Add `this.updateStatus(jobId, 'encoding')` before the FFmpeg encode step. Actually -- both capture and encode happen simultaneously in the piped approach, so keep the single `rendering` status for the entire capture+encode phase, then the mux is quick.
