  <action>
Create `src/lib/exportClient.ts` in the renderer project:

Define ExportRequest interface:
- settings: object containing all 17 settings fields (fps, scoreColor, scoreShadowDistance, hideUnplayedNotes, smoothReveal, scoreRegion, scoreBorder, scoreScale, musicFont, activeNoteheadColor, activeNoteheadScale, activeNoteheadEntryMs, activeNoteheadHoldMs, activeNoteheadExitMs, colorFullNote, audioDuration)
- syncAnchors: Map<string, number> -- the raw Map from syncStore (NOT pre-serialized)
- musicXmlContent: string -- the raw MusicXML text
- musicXmlFilename: string -- original filename for extension preservation
- audioFile: File -- the audio File object from the upload input
- bgImageFile?: File -- optional background image File

Define ExportResponse interface: { jobId: string; status: string }

Export async function requestExport(request: ExportRequest, backendUrl = 'http://localhost:3001'): Promise<ExportResponse>:
1. Create new FormData()
2. CRITICAL ORDERING: Append text fields FIRST, then files (busboy processes sequentially)
   - formData.append('settings', JSON.stringify(request.settings))
   - formData.append('syncAnchors', JSON.stringify(Object.fromEntries(request.syncAnchors)))
     NOTE: Object.fromEntries() converts Map to plain object -- this is the Map serialization fix
3. Append files SECOND:
   - formData.append('musicXml', new Blob([request.musicXmlContent], { type: 'application/xml' }), request.musicXmlFilename)
   - formData.append('audio', request.audioFile)
   - if (request.bgImageFile) formData.append('bgImage', request.bgImageFile)
4. const response = await fetch(`${backendUrl}/api/export`, { method: 'POST', body: formData })
   NOTE: Do NOT set Content-Type header -- browser auto-sets multipart boundary
5. const data = await response.json()
6. if (!response.ok) throw new Error(data.error || `Export failed: ${response.status}`)
7. return data as ExportResponse
