Update SyncEditor to consume `svgPages[]` from useVerovio and render stacked pages. SyncEditor is simpler than RegularRenderer -- it's user-scrolled (no camera), so it only needs visual page stacking, not page-offset Y computation.

**1. Update useVerovio destructuring:**

Change:
```typescript
const { svgString, toolkit, isLoading } = useVerovio(xml, containerWidth, 40);
```
To:
```typescript
const { svgPages, toolkit, isLoading } = useVerovio(xml, containerWidth, 40);
```

**2. Update the event extraction useEffect:**

Change the dependency from `svgString` to `svgPages`:
```typescript
useEffect(() => {
  if (svgPages.length === 0 || !osmdRef.current || !toolkit) return;

  requestAnimationFrame(() => {
    if (!osmdRef.current || !toolkit) return;
    const verovioSvg = osmdRef.current.querySelector('svg.definition-scale');
    if (!verovioSvg) return;
    // Pass single container -- SyncEditor doesn't need page-aware Y positions
    const extractedEvents = getEventsFromVerovio(toolkit, osmdRef.current);
    setEvents(extractedEvents);
  });
}, [svgPages, toolkit]);
```

**3. Update the animation controller exposure useEffect:**

Change `svgString` reference to `svgPages.length > 0`:
```typescript
useEffect(() => {
  if (!osmdRef.current || events.length === 0) {
    return;
  }
  // ... rest unchanged
}, [toolkit, svgPages, events.length, isRenderMode]);
```

Change the dependency array from `svgString` to `svgPages`.

**4. Update the JSX score rendering:**

Replace the single `dangerouslySetInnerHTML` div:

From:
```tsx
<div
  ref={osmdRef}
  dangerouslySetInnerHTML={svgString ? { __html: svgString } : undefined}
/>
```

To:
```tsx
<div
  ref={osmdRef}
  style={{ lineHeight: 0, fontSize: 0 }}
>
  {svgPages.map((svg, i) => (
    <div
      key={i}
      dangerouslySetInnerHTML={{ __html: svg }}
      style={{ lineHeight: 'normal', fontSize: 'initial' }}
    />
  ))}
</div>
```

Note: `lineHeight: 0` and `fontSize: 0` on the parent eliminates inline SVG gaps. Each child div resets these so text inside SVGs renders correctly. Alternatively, add a CSS rule to make SVGs display as block:

Actually, simpler approach -- just use CSS `display: block` on the SVGs. Add a `<style>` tag or inline style. The simplest approach for SyncEditor:

```tsx
<div ref={osmdRef}>
  {svgPages.map((svg, i) => (
    <div
      key={i}
      dangerouslySetInnerHTML={{ __html: svg }}
    />
  ))}
</div>
```

And add a CSS rule in the SyncEditor's style element (the one already created via `styleRef`):
```css
svg.definition-scale {
  display: block;
}
```

This ensures SVGs don't produce inline gaps. Add this to the existing `styleRef.current.innerHTML` assignment alongside the hover styles.
