  <name>Task 1: Create WebSocket progress route with state-sync, event forwarding, cancel handling, and heartbeat</name>
  <files>
    export-service/src/routes/progress.ts
  </files>
  <action>
    Create `export-service/src/routes/progress.ts` as a Fastify route plugin.

    **WebSocket route at `/export/:jobId/ws`:**

    ```typescript
    import type { FastifyInstance } from 'fastify';
    import type { WebSocket } from 'ws';
    import { jobManager } from '../jobs/jobManager.js';
    import type { JobProgressEvent } from '../jobs/types.js';
    ```

    Register the route with `{ websocket: true }` option. Handler signature: `(socket: WebSocket, req)`.

    **On connection:**

    1. Extract `jobId` from `req.params`.
    2. Look up job via `jobManager.getJob(jobId)`. If not found, send `{ type: 'error', error: 'Job not found' }`, close socket with code 4004, and return.
    3. Send immediate state-sync message for reconnection support:
       ```typescript
       socket.send(JSON.stringify({
         type: 'sync',
         status: job.status,
         stage: job.stage ?? null,
         frame: job.currentFrame ?? 0,
         totalFrames: job.totalFrames ?? 0,
         percent: job.percent ?? 0,
         error: job.error ?? null,
         downloadUrl: job.status === 'complete' ? `/api/export/${jobId}/download` : null,
       }));
       ```

    **Event forwarding:**

    4. Create a listener function `onProgress` that calls `socket.send(JSON.stringify(event))` only if `socket.readyState === socket.OPEN` (readyState check prevents "WebSocket is not open" errors).
    5. Subscribe: `jobManager.on(\`job:${jobId}\`, onProgress)`.

    **Cancel handling:**

    6. Listen for `socket.on('message', ...)`. Parse incoming message as JSON. If `msg.type === 'cancel'`, call `jobManager.cancelJob(jobId)`. Wrap parsing in try/catch to ignore malformed messages.

    **Heartbeat (30-second ping interval):**

    7. Set up a 30-second interval that calls `socket.ping()` if `socket.readyState === socket.OPEN`. Store the interval reference.
    8. Optionally track pong responses: on `socket.on('pong', ...)`, mark as alive. On each ping interval, if no pong since last ping, consider dead and terminate. However, for simplicity, just ping every 30s without liveness tracking (ws library handles pong automatically and most proxies will be satisfied by the ping traffic).

    **Cleanup on disconnect:**

    9. `socket.on('close', () => { ... })`:
       - Remove listener: `jobManager.off(\`job:${jobId}\`, onProgress)`
       - Clear heartbeat interval: `clearInterval(heartbeatInterval)`

    **Error handling:**

    10. `socket.on('error', () => { ... })`: Just log or ignore -- the close event follows and handles cleanup.

    Export as default async function following the existing route pattern (see `status.ts`).
  </action>

    **Step 1: Create download.ts**

    Create `export-service/src/routes/download.ts` as a Fastify route plugin.

    ```typescript
    import { createReadStream } from 'node:fs';
    import { stat } from 'node:fs/promises';
    import type { FastifyInstance } from 'fastify';
    import { jobManager } from '../jobs/jobManager.js';
    ```

    Register `GET /export/:jobId/download`:

    1. Extract `jobId` from `request.params`.
    2. Look up job via `jobManager.getJob(jobId)`. If not found, return 404 `{ error: 'Job not found' }`.
    3. If `job.status !== 'complete'` or `!job.outputPath`, return 409 `{ error: 'Export not complete' }`.
    4. Call `await stat(job.outputPath)` to get file size. If file does not exist (ENOENT), return 410 `{ error: 'Export file no longer available' }`.
    5. Create read stream: `const stream = createReadStream(job.outputPath)`.
    6. Return with manual headers (CRITICAL: cannot use reply.sendFile() because decorateReply is false):
       ```typescript
       return reply
         .header('Content-Type', 'video/mp4')
         .header('Content-Disposition', `attachment; filename="export-${jobId}.mp4"`)
         .header('Content-Length', fileStat.size)
         .send(stream);
       ```

    Export as default async function.

    **Step 2: Register @fastify/websocket and new routes in server.ts**

    Modify `export-service/src/server.ts`:

    1. Add import: `import websocket from '@fastify/websocket';`
    2. Add imports: `import progressRoutes from './routes/progress.js';` and `import downloadRoutes from './routes/download.js';`
    3. Register the WebSocket plugin BEFORE routes (after multipart, before route registrations):
       ```typescript
       await server.register(websocket);
       ```
    4. Register the new routes alongside existing ones:
       ```typescript
       await server.register(progressRoutes, { prefix: '/api' });
       await server.register(downloadRoutes, { prefix: '/api' });
       ```

    The order of route registration does not matter for Fastify (all routes are compiled into a radix tree at listen time), but register websocket plugin before routes so the `{ websocket: true }` option is available.
