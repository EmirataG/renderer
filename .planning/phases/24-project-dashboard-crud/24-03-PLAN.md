    - Add `export const dynamic = 'force-dynamic'`.
    - Fetch projects server-side:
      - Get session cookie, verify with `adminAuth.verifySessionCookie(session, true)`.
      - Query Firestore: `db.collection('projects').where('userId', '==', decoded.uid).orderBy('updatedAt', 'desc').get()`.
      - Map documents to `Project[]`, converting Firestore Timestamps to ISO strings: `doc.data().createdAt?.toDate().toISOString()`.
      - Wrap in try/catch; default to empty array on error.
    - Return `<Dashboard initialProjects={projects} />`.

    **2. Create `src/components/ProjectCard.tsx` (Client Component):**
    - `'use client'` directive.
    - Props: `project: Project`, `onDelete: (id: string, name: string) => void`.
    - Card layout using Tailwind:
      - Outer container: `group relative bg-neutral-900 border border-neutral-800 rounded-lg overflow-hidden hover:border-neutral-600 transition-colors cursor-pointer`.
      - Thumbnail area: `aspect-[4/3] bg-gradient-to-br from-neutral-800 to-neutral-900 flex items-center justify-center` with a music note SVG icon in neutral-700.
      - Metadata area: `p-4` with project name (truncated, `text-sm font-medium text-neutral-200`) and relative date (`text-xs text-neutral-500 mt-1`).
      - Format date: Use `new Date(project.updatedAt).toLocaleDateString(undefined, { month: 'short', day: 'numeric', year: 'numeric' })` for simple date display.
    - Three-dot menu button: `absolute top-2 right-2 opacity-0 group-hover:opacity-100 transition-opacity`. On click (with `e.stopPropagation()`), toggle a dropdown menu.
    - Dropdown menu: Simple absolute-positioned div with "Delete" option styled in red text. On click: call `onDelete(project.id, project.name)`. Click outside closes the dropdown (use a simple state toggle + blur or click handler).
    - Card click handler: `router.push(\`/project/${project.id}\`)` using `useRouter` from `next/navigation`. The three-dot button click uses stopPropagation to prevent card navigation.

    **3. Create `src/components/CreateProjectModal.tsx` (Client Component):**
    - `'use client'` directive.
    - Props: `isOpen: boolean`, `onClose: () => void`, `onCreated: (project: Project) => void`.
    - Two-step state: `step: 1 | 2`.
    - State: `scoreFile: File | null`, `audioFile: File | null`, `projectName: string`, `viewMode: 'page'` (always 'page' for now), `isCreating: boolean`.

    **Step 1 (Upload Files):**
    - Modal overlay: fixed inset-0 bg-black/60 backdrop-blur-sm z-50, centered content panel.
    - Panel: `bg-neutral-900 border border-neutral-800 rounded-xl p-6 max-w-lg w-full mx-4`.
    - Title: "New Project" with a step indicator (Step 1 of 2).
    - Two separate drag-and-drop zones side by side (or stacked on narrow screens):
      - **Score drop zone:** Accepts `.musicxml`, `.mxl`, `.mei`. Shows a music note icon and "Drop score file" text. On drop/click: validate file extension, store in `scoreFile` state. Show filename when file is selected with a small X button to remove.
      - **Audio drop zone:** Accepts `.mp3`, `.wav`. Shows a speaker/audio icon and "Drop audio file" text. Similar behavior.
    - Each drop zone: border-2 border-dashed, changes border color on drag-over, shows filename after upload.
    - File validation: Check extension against allowed types. Show error toast for invalid files.
    - "Next" button: enabled only when both scoreFile AND audioFile are set. Advances to step 2.
    - "Cancel" button: calls onClose, resets state.

    **Step 2 (Project Details):**
    - Same modal container.
    - Title: "New Project" with step indicator (Step 2 of 2).
    - Project name input: `<input type="text">` with placeholder "Project name", required. Styled with Tailwind: `w-full bg-neutral-800 border border-neutral-700 rounded-lg px-4 py-2.5 text-neutral-200 placeholder-neutral-500 focus:outline-none focus:border-neutral-500`.
    - View mode cards (per user locked decision):
      - Two cards side by side:
        - **"Page view"** card: border-2 border-blue-500/60 bg-blue-500/10 (active state). Shows checkmark or radio indicator. Text: "Page view".
        - **"Single line"** card: border-2 border-neutral-700 bg-neutral-800/50 opacity-50 cursor-not-allowed. Text: "Single line" with a small "Coming soon" badge/label below or beside it.
    - "Back" button: returns to step 1, preserving file selections.
    - "Create" button: enabled only when `projectName.trim()` is non-empty. On click:
      - Set `isCreating: true`.
      - POST to `/api/projects` with `{ name: projectName.trim(), viewMode: 'page' }`.
      - On success: read the `{ id }` from response. Construct a `Project` object with the ID, name, viewMode, and current timestamp strings. Call `onCreated(project)`.
      - Then navigate to `/project/${id}` using `router.push`.
      - IMPORTANT: The actual file content (score XML, audio blob) stays in browser memory. It is NOT uploaded to a server. The existing UploadDropZone in the editor will handle file loading for now. Files need to be re-uploaded each session until file persistence is added in Phase 25.
      - On error: show error toast, set `isCreating: false`.

    **4. Create `src/components/Dashboard.tsx` (Client Component):**
    - `'use client'` directive.
    - Props: `initialProjects: Project[]`.
    - State: `projects` (initialized from `initialProjects`), `isCreateModalOpen: boolean`.
    - Import `useToast` from `@/hooks/useToast`. NOTE: The Dashboard is rendered inside `page.tsx` which is a Server Component. The Dashboard itself is a client component, but Toast context must wrap it. Update `page.tsx` to wrap Dashboard in `<ToastProvider>` (imported from `@/components/Toast`).
    - Import `useRouter` from `next/navigation`, `signOut` from `firebase/auth`, `auth` from `@/lib/firebase-client`.

    **Header area:**
    - App title/logo: "Manuscript" or similar.
    - "New Project" button: opens the create modal. Styled as a primary action button.
    - Sign out button in the top right (same pattern as editor).

    **Grid area:**
    - If `projects.length === 0`: render empty state -- centered content with music note icon, "No projects yet" heading, description text, and a "New Project" button.
    - Otherwise: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-6` with ProjectCard for each project.

    **Create callback:**
    - `handleCreated(project: Project)`: add the new project to the front of the projects array in state.

    **Delete flow (per user locked decisions):**
    - `handleDelete(projectId: string, projectName: string)`:
      1. Store the deleted project reference.
      2. Remove the project from local state immediately (optimistic).
      3. Set up a timeout (5 seconds) for the actual DELETE API call.
      4. Show toast: message `"Project deleted"`, type `"info"`, with action `{ label: "Undo", onClick: undoFn }`, duration `5000`.
      5. The `undoFn`: clear the timeout, restore the project to local state.
      6. On timeout expiry: fire `fetch(\`/api/projects/${projectId}\`, { method: 'DELETE' })`.
      7. Handle unmount cleanup: in a useEffect cleanup or useRef, if the component unmounts with a pending delete timeout, fire the DELETE immediately (don't let deleted projects reappear).
    - Before deletion, show a confirmation dialog: simple modal or use window.confirm for simplicity. Message: `"Delete '${projectName}'? This cannot be undone."` Per user decision this is a confirmation dialog, not just the toast. Use a small custom dialog component (inline in Dashboard or a simple confirm):
      - A small overlay dialog with the confirmation message, "Cancel" button, and "Delete" button (red). This is NOT `window.confirm` -- it should match the app's dark theme.
      - State: `deleteConfirm: { id: string; name: string } | null`. When set, show the dialog. On confirm, proceed with the delete flow above.

    **CreateProjectModal integration:**
    - Render `<CreateProjectModal isOpen={isCreateModalOpen} onClose={() => setIsCreateModalOpen(false)} onCreated={handleCreated} />`.

    **Sign out handler:**
    - Same pattern: DELETE /api/auth/session, signOut(auth), router.push('/login').
